# Security Policy

## Supported Scope

Security reports are most helpful when they affect:

- published `evidpath` packages
- the GitHub Actions release flow
- provider credential handling
- reference service or example service behavior that could expose sensitive data

## Reporting a Vulnerability

Please do not open public issues for suspected security problems.

Instead, report privately through GitHub Security Advisories when available, or contact the maintainer privately through the GitHub profile:

- <https://github.com/AlankritVerma01>

Include:

- a short description of the issue
- affected files, commands, or workflows
- reproduction steps if available
- any suggested mitigation

We will acknowledge the report, investigate, and coordinate a fix before public disclosure when appropriate.
